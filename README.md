# Mao Method Skill

> [!IMPORTANT]
> 这不是“毛选金句生成器”。
> 这是把 **《毛泽东选集》蒸馏成一套现实问题拆解工具** 的 skill：**先把问题问清，再把结构拆开，最后才分析和给方案。**

> [!TIP]
> 如果你只是想听两句像样的狠话，网上很多。  
> 如果你真想知道这局到底卡在哪、先动谁、动什么、别踩哪颗雷，用这个。

## 它到底卖什么

- **蒸馏源头很明确**：不是泛泛而谈的“方法论”，是从 **《毛泽东选集》** 里抽出来、能落地的分析方法。
- **不抢答**：不会一上来就装懂。会先追问关键缺口，把问题问到基本成形，再进入正式分析。
- **不只会讲道理**：最后给的是结构判断、路线选择、风险边界、下一步动作。
- **能出成品**：除了文字版深度分析，还能按需生成一份可分享的 **单文件 HTML 报告**。

## 它适合拆什么

它不只适合团队问题。恰恰相反，越是那种“表面像小摩擦，背后其实一堆结构卡着”的事，它越有用。

比如：

- 项目推进不动，大家都很忙，但结果就是不动。
- 合伙人、同事、上下级之间互相拉扯、推诿、甩锅、卡信息。
- 男女关系里表面是吵架，背后可能还牵着第三方、父母、旧账、边界、资源、信任。
- 团队看起来像执行问题，其实是路线没收口、阶段没看对、控制点不在纸面关系上。
- 你想换工作、分手、止损、继续谈、还是直接掀桌，但脑子里一锅粥。

一句话：

**它擅长的不是“答题”，而是“拆局”。**

## 它怎么工作

不是“你一问，它就一顿输出”。  
正常流程是这样的：

1. **先问清**：目标是什么，最近哪件事最说明问题，关键人物是谁，你已经试过什么，现在真正的约束是什么。
2. **再拆结构**：看主要卡点、阶段、关系、资源、边界、控制点到底落在哪。
3. **最后再分析**：给出核心判断、路线建议、风险提醒和下一步动作。
4. **正式输出前再确认形式**：你要文字版，还是要 HTML 报告。

也就是说，它默认不是“我很聪明所以我先说”，而是：

**先别急着下结论，先把题目看对。**

## 你可以怎么触发它

这些说法都可以：

- `用毛选帮我分析这个项目为什么推进不动`
- `用教员的方法帮我拆一下我和合伙人的关系`
- `按主要矛盾和阶段判断，分析我现在该不该换工作`
- `用毛选的方法看看，我和女朋友这事到底卡在哪`
- `最后请给我一份 HTML 报告`

如果你想让结果更准，最好顺手带上这五样：

- `目标`：你到底想推进什么结果。
- `事件`：最近一次最能说明问题的关键事件。
- `人物`：关键人物、第三方、关系人是谁。
- `尝试`：你已经做过什么。
- `约束`：你现在最真实的限制和底线。

## 最后会给你什么

### 1. 文字版深度分析

适合先把局面看明白。通常会包括：

- 问题重述
- 核心判断
- 当前阶段
- 推荐路线
- 风险提醒
- 下一步动作

### 2. HTML 报告

适合你要一份成品，方便保存、复盘、转发。  
复杂问题还可以带上：

- 关系图
- 时间线
- 路线比较
- 证据链
- 控制点 / 权力点分布
- 执行计划

## 不适合什么

- 只想摘毛选原文。
- 只想拿“主要矛盾”“统一战线”这些词去压人。
- 问题其实很轻，普通建议就够了。
- 你要的是纯技术细节，不是结构分析。

> [!WARNING]
> 这个 skill 最烦的一件事，就是把方法论玩成“大词压人”。
> 真要分析，就先把局面拆清楚；别上来就给自己封个总指挥。
