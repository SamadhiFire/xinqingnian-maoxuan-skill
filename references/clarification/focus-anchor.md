# 主问题锚点与轮次日志规范

## 一、文件作用

这个文件只解决一件事：

- 在长问题、复杂问题、多轮问题里，防止分析对象漂移，避免模型“越聊越多，越聊越偏”，最后答不到用户最想问的点。

它不负责：

- 替代 `ambiguity-gate.md` 判断是否要澄清
- 替代 `problem-restatement.md` 做正式问题重述
- 直接决定方法卡怎么选
- 直接代替最终正式输出

一句话说：

- 不要把“记住主问题”寄托在模型记性上，而要把主问题外部化、显性化、反复校准。

## 二、核心原则

复杂长问题里，最危险的不是信息少，而是：

- 戏剧性细节太多，把主问题盖掉
- 新补充信息很多，挤掉最初目标
- 一边澄清一边扩题，最后分析对象变了
- 用户明明问了 `3` 到 `6` 个具体问题，输出却变成一篇“看起来很完整”的长文

所以默认原则是：

1. 先冻结当前主问题
2. 再限定本轮只回答哪 `1` 到 `2` 个问题
3. 每轮结束更新一次进度
4. 正式输出时，回对用户原始问题，而不是只对当前叙事惯性负责

## 三、什么时候必须启用

满足下面任一情况，就应启用主问题锚点机制：

- 问题复杂度达到 `L3` 或 `L4`
- 涉及三方及以上关键人物或组织
- 用户一次性抛出 `3` 个及以上明确问题
- 材料很长，且有明显时间线、关系线、资源线并行
- 已经进入第二轮及以上澄清
- 用户明确期待 `HTML 报告`、正式报告或可保存成品

普通轻问题通常不必显式启用。

## 四、主问题锚点卡

一旦启用，就先建立一张简短的 `主问题锚点卡`。  
这张卡不是正式分析，也不是报告正文，而是整个后续分析的“题眼固定器”。

至少包含下面 5 项：

1. 用户当前最想解决的一句话问题
2. 当前时间窗口或目标时点
3. 当前最硬的约束或不能承受的代价
4. 本轮准备优先回答的 `1` 到 `2` 个问题
5. 当前明确不展开或暂不裁决的内容

推荐格式：

```text
主问题锚点：
- 这次真正要解决的是……
- 当前时间窗口是……
- 最硬约束是……
- 这轮我先只回答……
- 这些内容先不展开……
```

## 五、轮次日志

有了主问题锚点后，每一轮实质性回复结束时，都应做一次轻量更新。  
目的不是写工作日志，而是防止分析对象偷偷漂移。

最小要更新 4 项：

1. 目前已确认
2. 这轮已经回答了什么
3. 还没回答什么
4. 仍待确认哪些信息会改写判断

推荐格式：

```text
目前进度：
- 已确认：……
- 这轮已回答：……
- 还没回答：……
- 仍待确认：……
```

如果用户已经给了编号问题，还应补一行：

```text
- 原始问题对照：1 已答 / 2 部分回答 / 3 待答
```

## 六、问题重述和锚点卡的区别

两者都要“收束问题”，但作用不同：

- `主问题锚点卡`：冻结题眼，防止后面越走越偏
- `问题重述`：把材料整理成可分析版本，准备进入正式分析

不要只做问题重述，不做锚点。  
对复杂长问题来说，只有重述，没有锚点，仍然很容易在后续长文中丢题。

## 七、每轮开始前的漂移检查

在写任何一段较长分析前，先快速过下面 5 个问题：

1. 我现在准备写的这段，服务的是用户哪个原始问题？
2. 新信息改变的是主问题，还是只是补证据？
3. 我是不是被一个很有戏剧性的细节带偏了？
4. 我现在写的是事实、推断、传闻还是未确认缺口？
5. 这轮结束后，用户能不能一眼看出“我到底答到了哪里”？

只要上面有 `2` 项以上答不上来，就先回锚点卡，不要继续铺长文。

## 八、正式输出时怎么防迷失

正式输出时，默认按下面顺序：

1. 先回扣主问题锚点
2. 再按用户原始问题顺序回答
3. 如果某题仍不能稳答，明确标成“部分回答”或“仍待确认”
4. 最后才放补充说明、方法出处或扩展观察

如果用户原始输入里有编号问题，最终输出不要打乱顺序另起一套目录。  
最稳的做法是沿用用户原编号。

## 九、要不要写外部日志文件

默认不要把“写日志文件”当成主解法。

原因是：

- 不是所有平台都允许稳定读写文件
- 不是所有模型都会可靠回看文件
- 文件日志可能增加用户的额外负担和隐私顾虑
- 如果主问题锚点没有在对话里显性化，光写文件也仍然会漂移

所以优先级是：

1. 对话内主问题锚点
2. 对话内轮次日志
3. 最终输出对照用户原问题
4. 如平台支持、且用户愿意，再把锚点同步到外部日志文件

外部日志只能做辅助，不能代替前面三项。

## 十、禁止事项

- 不要默认相信模型会“自然记住主问题”
- 不要在复杂长问题里，一边补信息一边不断扩题
- 不要把最戏剧化的细节自动当成最核心的问题
- 不要最终写成一篇看似完整、却没有逐项回应用户问题的长文
- 不要把外部日志文件当成唯一防迷失手段
